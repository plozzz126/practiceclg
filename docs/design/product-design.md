# DevLink Product Design

## Product direction

DevLink is not a landing page product. It should feel like a practical student workspace where people can:

- discover projects by stack and direction
- evaluate whether they fit a team
- apply with context
- review teammates through profile, skills, and deadlines

## Visual direction

- Dark workspace UI with teal as the main action color
- Dense but readable cards and panels
- Desktop left rail, mobile bottom navigation
- Strong emphasis on stack tags, role tags, and deadline visibility
- Minimal decorative noise so the product still feels like a tool

## Core surfaces

1. Auth
2. Dashboard
3. Project catalog
4. Project detail workspace
5. Teammate catalog
6. Profile setup

## Interaction principles

- Search and filters should be visible without hunting through menus.
- Project cards should answer the first three questions fast: what is it, what stack is needed, and how urgent is it.
- Join requests should surface match score, skills, role fit, and motivation in one place.
- Mobile should preserve the same flows, not a stripped-down placeholder version.
